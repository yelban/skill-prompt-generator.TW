## ⚡ 快速選擇指南

### 決策樹

```
你想生成什麼？
       │
       ├─ 有人物嗎？
       │     │
       │     ├─ YES ─────────────→ intelligent-prompt-generator
       │     │                      （除非強調藝術技法術語）
       │     │
       │     └─ NO ─→ 主體是什麼？
       │              │
       │              ├─ 風景/靜物/藝術繪畫 → art-master
       │              ├─ 產品/商品/電商圖 → product-master
       │              ├─ 海報/PPT/UI設計 → design-master
       │              └─ 影片/動畫/運鏡 → video-master
       │
       └─ 需要完整設計系統（模板）？
             │
             ├─ Apple 風格 → design_templates (apple_*)
             ├─ Google 風格 → Material Design 模板
             ├─ 微軟風格 → Fluent Design 模板
             └─ 手繪風格 → 手繪資訊圖模板
```

### 一句話選擇

| 你的需求 | 使用的 Skill |
|---------|-------------|
| 人物肖像、人像攝影 | `intelligent-prompt-generator` |
| 山水風景、藝術畫作 | `art-master` |
| 海報、PPT、UI 設計 | `design-master` |
| 產品攝影、電商圖片 | `product-master` |
| 影片場景、運鏡效果 | `video-master` |
| 學習新風格、擴充元素庫 | `universal-learner` |
| 分析已生成的提示詞 | `prompt-analyzer` |
| 從提示詞提取可複用元素 | `prompt-extractor` |
| 逆向工程，提取技巧知識 | `prompt-xray` |

## 🎭 領域專家層（5大生成器）

### 1️⃣ intelligent-prompt-generator（核心引擎）

**定位**: 最智慧的生成器，支援三種模式

| 屬性 | 說明 |
|------|------|
| **元素數** | 502（portrait）+ 跨域呼叫 1,246 全部 |
| **核心能力** | 語義理解、常識推理、一致性檢查 |
| **三種模式** | Portrait / Cross-Domain / Design |

#### ✅ 適用場景

| 場景 | 說明 |
|------|------|
| 人物肖像 | 五官、妝容、表情、姿勢 |
| 複雜場景 | 人物+動作+特效（自動跨域） |
| 需要智慧推理 | 如「亞洲女性」自動推理典型特徵 |

#### 🎨 支援風格

| 風格類型 | 範例 |
|---------|------|
| 導演風格 | 張藝謀（戲劇光影）、徐克（武俠飄逸）、王家衛（懷舊氛圍） |
| 時代風格 | 古裝、民國、現代、未來 |
| 視覺風格 | 寫實、動漫、電影級、賽博朋克 |
| 光影風格 | 自然光、電影光、霓虹光、柔光、戲劇光 |

#### 📝 使用範例

```
# Portrait 模式（純人像）
生成電影級亞洲女性肖像，張藝謀電影風格

# Cross-Domain 模式（自動跨域）
生成龍珠悟空打出龜派氣功的蠟像3D感

# Design 模式（設計系統）
生成溫馨可愛風格的兒童教育海報

# 古裝人像
生成仙劍奇俠傳真人電影風格的年輕古裝女子

# 賽博朋克
生成賽博朋克風格的動漫少女提示詞
```

#### 🔧 進階功能

**常識推理**：自動根據人種推理眼睛顏色、髮色
```
東亞人 → 黑/深棕眼睛、黑髮
歐洲人 → 藍/綠/棕/灰眼睛、多種髮色
```

**一致性檢查**：自動修正衝突
```
東亞人 + 綠眼睛 → 自動修正為棕色眼睛
古裝 + 現代妝容 → 自動修正為傳統妝容
```

## 🔧 輔助工具層（6個工具）

### universal-learner（通用學習器）

**功能**: 從任何領域的提示詞中自動提取可複用元素

| 使用時機 | 操作方式 |
|---------|---------|
| 學習新風格 | `學習這個 Prompt: [貼上完整 Prompt]` |
| 批次學習 | `批次學習所有 Prompts` |
| 學習設計系統 | `學習這個設計系統：[完整規範]` |

**輸出**: 學習報告 + 元素自動存入資料庫

---

### prompt-extractor（元素提取器）

**功能**: 從海量提示詞中提煉可複用的模組元件

| 使用時機 | 操作方式 |
|---------|---------|
| 分析優秀提示詞 | `使用 prompt-extractor 分析 my_prompts.txt` |
| 直接貼上 | 直接貼上提示詞文字 |

**輸出**:
- `extracted_modules.json` - 完整提取結果
- `module_library.json` - 去重後的模組庫
- `analysis_report.md` - 分析報告（含學習卡片）

**提取的 10 大模組類型**:
1. 主體變數 (Subject Variables)
2. 視覺風格 (Visual Style)
3. 技術參數 (Technical Parameters)
4. 細節增強 (Detail Enhancers)
5. 情緒氛圍 (Mood & Atmosphere)
6. 約束條件 (Constraints)
7. 構圖參數 (Composition)
8. 色彩方案 (Color Scheme)
9. 時間/季節 (Time & Season)
10. 參考藝術家 (References)

---

### prompt-xray（提示詞逆向工程）

**功能**: 從優秀提示詞中逆向提取「如何做 X」的知識

| 使用時機 | 操作方式 |
|---------|---------|
| 提取顏色知識 | `從已分析的提示詞中，提取「如何控制顏色」的知識` |
| 提取佈局知識 | `提取「如何控制空間佈局」的知識` |
| 構建完整知識庫 | `從已分析的提示詞中，構建完整知識庫` |

**可提取的知識維度**:
- 如何控制顏色？（配色公式、關鍵詞庫）
- 如何控制空間佈局？（視角、構圖規則）
- 如何新增標誌性符號？（文字語法、Logo）
- 如何控制材質？（表面特徵、物理屬性）
- 如何控制光影？（光源類型、布光方案）

---

### prompt-analyzer（提示詞分析器）

**功能**: 分析已生成的提示詞，提供洞察和推薦

| 功能 | 操作方式 |
|------|---------|
| 檢視詳情 | `檢視 Prompt #5 的詳情` |
| 對比分析 | `對比 Prompt #5 和 #17` |
| 相似推薦 | `推薦與 #5 相似的 Prompt` |
| 元素統計 | `元素庫有哪些類別？` |
| 風格推薦 | `古裝風格應該用什麼元素？` |

---

### domain-classifier（領域分類器）

**功能**: 智慧判斷提示詞的領域歸屬

**自動路由**: 系統內部使用，自動判斷使用者需求屬於哪個領域

---

## 💡 使用技巧

### 1. 選擇正確的 Skill

- **有人物** → 優先 `intelligent-prompt-generator`
- **純藝術/風景** → `art-master`
- **產品** → `product-master`
- **設計/海報** → `design-master`
- **影片** → `video-master`

### 2. 善用跨域模式

當需求複雜時（人物+動作+特效），系統會自動啟用 Cross-Domain 模式，組合多個領域的元素。

### 3. 模板 vs 元素

- **模板級**: 需要完整設計系統時（如「Apple 風格 PPT」）
- **元素級**: 需要靈活組合時（如「Bento Grid + 玻璃態」）

### 4. 善用學習工具

- 遇到優秀的提示詞 → `prompt-extractor` 提取可複用元素
- 想知道「如何做 X」 → `prompt-xray` 逆向提取知識
- 想分析已生成的結果 → `prompt-analyzer` 對比分析
